🚀 RAILWAY DEPLOYMENT CHECKLIST
═══════════════════════════════════════════════════════════════

WHY TRADES DON'T WORK ON RAILWAY:
Railway doesn't have your Telegram session file (session_testpob1234.session)
Solution: Use string session stored in environment variable

═══════════════════════════════════════════════════════════════
STEP-BY-STEP FIX:
═══════════════════════════════════════════════════════════════

[ ] 1. GENERATE STRING SESSION (Run locally)
    Command: python generate_string_session.py
    - Enter OTP when prompted
    - Copy the TELEGRAM_STRING_SESSION value

[ ] 2. ADD TO RAILWAY VARIABLES
    Go to: https://railway.app/dashboard
    - Click your project
    - Click "Variables" tab
    - Add these variables:

    ┌─────────────────────────────────────────────────────────┐
    │ TELEGRAM_STRING_SESSION = <paste_from_step_1>           │
    │                                                          │
    │ TELEGRAM_API_ID = 28375707                              │
    │ TELEGRAM_API_HASH = cf54e727df04363575f8ee9f120be2c9    │
    │ TELEGRAM_PHONE = +12427272924                           │
    │ TELEGRAM_CHANNEL = testpob1234                          │
    │                                                          │
    │ SSID_DEMO = <your_demo_ssid>                            │
    │ SSID_REAL = <your_real_ssid>                            │
    │                                                          │
    │ TRADE_AMOUNT = 1                                        │
    │ IS_DEMO = True                                          │
    │ MULTIPLIER = 2.5                                        │
    │ MARTINGALE_STEP = 0                                     │
    └─────────────────────────────────────────────────────────┘

[ ] 3. PUSH CODE TO GITHUB
    git add .
    git commit -m "Add Railway string session support"
    git push

[ ] 4. VERIFY DEPLOYMENT
    Check Railway logs for:
    ✓ "Using string session (Railway deployment)"
    ✓ "Connected to: Test"
    ✓ "Pre-connected to PocketOption"
    ✓ "Waiting for signals..."

[ ] 5. TEST TRADING
    - Send test signal to Telegram channel
    - Check Railway logs for:
      ✓ "NEW MESSAGE RECEIVED"
      ✓ "Trade signal received"
      ✓ "Order placed!"

TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════

Problem: "Invalid string session"
Fix: Regenerate with python generate_string_session.py

Problem: "SSID expired"
Fix: Get new SSID from PocketOption (F12 > Network > WebSocket)
     Update SSID_DEMO or SSID_REAL in Railway variables

Problem: "Channel not found"
Fix: Verify TELEGRAM_CHANNEL matches your channel username

Problem: "Balance data not available"
Fix: This is normal - bot will still place trades
